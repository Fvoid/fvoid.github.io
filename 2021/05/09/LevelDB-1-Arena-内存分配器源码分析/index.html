2. C++ & Math Prep

2.1 = delete

Arena(const Arena&) = delete

= delete is to prevent compiler using Arena::Arena(const Arena&) and Arena& Arena::operator=(const Arena&) by default.

2.2 sizeof(void*)

  • If the system is 16-bit, size of void pointer is 2 bytes.
  • If the system is 32 bit, size of void pointer is 4 bytes.
  • If the system is 64 bit, size of void pointer is 8 bytes.

2.3 reinterpret_cast<uintptr_t>

The type corresponds to the size of the machine pointer

2.4 mod calculation

3. Main Interface

Method Public Explanation
char* Arena::Allocate(size_t bytes) True common way to allow memory
char* Arena::AllocateAligned(size_t bytes) True allocate memory in align
char* Arena::AllocateFallback False allocate additional memory in new block
char* Arena::AllocateNewBlock False create new block to memory pool

The return value of all these functions is the memory starting address.

4. Source Code Analysis

  • function Arena::Allocate

    • allocate bytes size should larger than 0
    • when byte size fits in current block
      • mark current empty memory address -> result
      • increment empty memory address ptr
      • substract remaining bytes counter
    • otherwise, fallback allocate bytes

  • function Arena::AllocateFallback

    • if input_byte is larger than 1/4 of kBlockSize (4KB)
      • allocate a memory in input_byte size as a new block
    • else
      • allocate a new 4KB block and waste the remaining space

  • function Arena::AllocateNewBlock

    • new input_byte size
    • push new memory starting address to vector of blocks
    • cumulative add up memory usage

  • function Arena::AllocateAligned

    • get system pointer size
    • get current alloc_ptr mod system_pointer size
    • calculate needed = bytes + mod_result
    • when needed fits in current block
      • get memory address of current block
    • otherwise, allocate the bytes into a new block by fallback
